# 一次项目复盘的写作模板

项目复盘不一定要写得很复杂。对个人博客来说，最重要的是把当时的判断、取舍和结果记录下来。

## 可以包含的内容

1. 项目背景：为什么要做这个项目；
2. 核心目标：希望解决什么问题；
3. 技术实现：用了哪些技术和方案；
4. 遇到的问题：哪些地方比预期困难；
5. 后续改进：如果重新做，会怎样调整。

这类文章可以帮助自己沉淀经验，也能让项目展示页更有说服力。
